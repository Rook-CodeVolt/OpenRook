# Implementation Playbook (Any Org Size)

## Phase 1 — Establish baseline governance
1. Assign accountable owner and governance forum.
2. Build initial AI/agent inventory.
3. Define risk tiers and autonomy boundaries.
4. Publish policy baseline and exception workflow.

**Exit criteria**
- owner assigned,
- inventory >= 90% known systems,
- policies approved,
- exception path operational.

## Phase 2 — Implement foundational controls
1. Enforce RBAC + least privilege.
2. Configure environment segregation.
3. Enable tamper-evident logs and alerting.
4. Stand up backup/recovery tests.

**Exit criteria**
- access review completed,
- prod separation verified,
- logs and alerts validated,
- recovery test passed.

## Phase 3 — Pilot low-risk use cases
1. Select pilot scope with clear KPIs.
2. Run threat/risk assessment.
3. Execute staged rollout with rollback readiness.
4. Collect evidence and stakeholder feedback.

**Exit criteria**
- KPI baseline captured,
- no unresolved critical findings,
- rollback validated,
- pilot review signed off.

## Phase 4 — Scale safely
1. Expand to additional use cases/teams.
2. Introduce autonomy tiers + approval automation.
3. Add budget/concurrency governors.
4. Run periodic control and incident drills.

**Exit criteria**
- governance coverage expanded,
- approval compliance high,
- stable reliability/cost profile.

## Phase 5 — Optimise and learn
1. Promote recurring fixes into reusable artefacts.
2. Tune model routing and policies.
3. Retire ineffective controls/models.
4. Report outcomes quarterly.

**Exit criteria**
- measurable efficiency gains,
- reduced repeat incidents,
- improved time-to-resolution.
